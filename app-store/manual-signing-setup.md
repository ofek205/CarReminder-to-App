# מדריך הקמה חד פעמי לחתימה ידנית של אפליקציית האייפון

מדריך זה מסביר כיצד להפיק תעודת הפצה ופרופיל חתימה לחנות האפליקציות, להמיר אותם לפורמט שמתאים להעלאה כסודות בריפו, ולהפעיל את תהליך הבנייה האוטומטי במכונת הבנייה של הריפו. כל הפעולות מתבצעות ממחשב הוויינדוס שלך, ללא צורך במק.

---

## דרישות מקדימות

- חשבון מפתח אפל פעיל ומשולם
- הרשאת מנהל בחשבון
- מזהה הצוות
```
L36CBSRNZT
```
- מזהה החבילה
```
com.carreminders.app
```
- כלי שורת פקודה זמין במחשב, מגיע יחד עם התקנת גיט עבור ויינדוס
```
openssl
```

בכל הפקודות למטה אנו מניחים שאתה עובד בתיקיית עבודה ייעודית. צור אותה ונווט אליה דרך הפאוורשל
```powershell
mkdir $HOME\ios-signing
cd $HOME\ios-signing
```

---

## שלב 1 — יצירת בקשת חתימה במחשב הוויינדוס

ראשית מייצרים מפתח פרטי בקובץ נפרד, ואז יוצרים בקשת חתימה שמתבססת עליו. שמור את שני הקבצים, את המפתח הפרטי תצטרך גם בשלב ההמרה לקובץ ארוז.

הפעל בפאוורשל
```powershell
openssl genrsa -out ios_distribution.key 2048
openssl req -new -key ios_distribution.key -out ios_distribution.csr -subj "/emailAddress=YOUR_EMAIL@example.com/CN=CarReminder Distribution/C=IL"
```

החלף את כתובת הדואר במחרוזת לכתובת שלך בחשבון אפל.

לבדיקה שהבקשה תקינה הפעל
```powershell
openssl req -in ios_distribution.csr -noout -text
```

הפלט אמור להציג בלוק שמתחיל ב
```
Certificate Request
```
ולהראות את שם המשתמש והכתובת שהקלדת. אם רואים את זה — הבקשה תקינה.

## שלב 2 — העלאת הבקשה לאפל וקבלת תעודת הפצה

פתח את עמוד ניהול התעודות בלוח הבקרה של מפתחי אפל
```
https://developer.apple.com/account/resources/certificates/add
```

בחר את סוג התעודה
```
Apple Distribution
```

לחץ על המשך, ובמסך העלאת הבקשה גרור את הקובץ
```
ios_distribution.csr
```

לחץ על המשך, ואז על הורדה. אפל תשלח אליך קובץ בשם
```
distribution.cer
```

שמור אותו לתיקיית העבודה לצד המפתח הפרטי.

---

## שלב 3 — המרת התעודה והמפתח הפרטי לקובץ ארוז

אפל מחזירה תעודה בפורמט בינארי. כדי שמכונת הבנייה תוכל לייבא אותה לצביר המפתחות, צריך לארוז אותה יחד עם המפתח הפרטי לקובץ אחד מוגן בסיסמה.

הפעל את שלוש הפקודות הבאות לפי הסדר
```powershell
openssl x509 -inform DER -in distribution.cer -out distribution.pem -outform PEM
openssl pkcs12 -export -legacy -out ios_distribution.p12 -inkey ios_distribution.key -in distribution.pem
```

הפקודה השנייה תבקש להגדיר סיסמת ייצוא. בחר סיסמה חזקה ושמור אותה — תזדקק לה גם בהמשך כסוד ברירת מחדל בריפו.

בקצה הפקודה תראה
```
Mac Verify OK
```
או פלט שקט אם הצליח. הקובץ
```
ios_distribution.p12
```
הוא הקובץ הארוז שתעלה לריפו בשלב מאוחר יותר.

הערה — הדגל
```
-legacy
```
חיוני, כי גרסאות חדשות של הכלי משתמשות באלגוריתם שאינו תואם לצביר המפתחות של מק מהמכונה הוירטואלית. אם תשמיט את הדגל, הייבוא במכונה הוירטואלית ייכשל בלי הודעת שגיאה ברורה.

## שלב 4 — רישום מזהה האפליקציה בלוח הבקרה

פתח את עמוד ניהול המזהים
```
https://developer.apple.com/account/resources/identifiers/list
```

אם המזהה כבר קיים — דלג לשלב הבא. אחרת, לחץ על הוספה, בחר במסך הסוג
```
App IDs
```
ובמסך הבא בחר
```
App
```

בשדה התיאור הקלד שם קריא, לדוגמה
```
CarReminder
```

בשדה המזהה בחר במצב מפורש והקלד בדיוק
```
com.carreminders.app
```

בחלק היכולות אין צורך לסמן דבר — השאר את ברירות המחדל. לחץ על המשך ועל אישור.

## שלב 5 — יצירת פרופיל חתימה לחנות האפליקציות

פתח את עמוד הוספת הפרופילים
```
https://developer.apple.com/account/resources/profiles/add
```

בחר את סוג הפרופיל
```
App Store
```

לחץ על המשך, ובחר את מזהה האפליקציה
```
com.carreminders.app
```

לחץ על המשך, ובחר את התעודה שהונפקה בשלב השני. השם שלה יופיע ככזה שמתחיל ב
```
Apple Distribution
```
עם תאריך הנפקה של היום.

לחץ על המשך. בשדה שם הפרופיל הקלד בדיוק את המחרוזת הבאה — אות בגודל עברי, רווחים בדיוק כפי שמופיעים, בלי תווים נוספים
```
CarReminder App Store
```

חשוב — שם זה חייב להיות זהה במאת האחוזים למה שמופיע במאגר במקובץ ההגדרות של האקסקוד. אם תקליד אחרת, הבנייה במכונה הוירטואלית תיכשל עם הודעה שמדברת על אי התאמת פרופיל.

לחץ על יצירה ועל הורדה. תקבל קובץ עם הסיומת
```
.mobileprovision
```

הקובץ הזה אינו צריך לעלות לריפו — הוא יורד אוטומטית במכונה הוירטואלית באמצעות מפתח ה
```
API
```
שכבר קיים בסודות. שמור עותק מקומי בכל זאת לתיקיית העבודה, למקרה שתצטרך לחתום בעתיד מהמחשב המקומי.

## שלב 6 — המרת הקובץ הארוז למחרוזת בייסים שישים וארבע

הסודות בריפו מאחסנים מחרוזות טקסט בלבד, ולא קבצים בינאריים. לכן צריך לקודד את הקובץ הארוז בקידוד בייסים שישים וארבע.

הפעל בפאוורשל
```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("$HOME\ios-signing\ios_distribution.p12")) | Set-Clipboard
```

מחרוזת הקידוד נמצאת כעת בלוח ההעתקה של ויינדוס. אל תפתח אותה בעורך טקסט לפני ההדבקה — בעורכים מסויימים יש שבירת שורות שתשבש את הסוד.

## שלב 7 — הוספת שלושה סודות חדשים בריפו

פתח את עמוד הסודות של הריפו
```
https://github.com/ofek205/CarReminder-to-App/settings/secrets/actions
```

לחץ על
```
New repository secret
```
שלוש פעמים, וצור את שלושת הסודות הבאים בזה אחר זה.

הסוד הראשון — תוכן הקובץ הארוז כמחרוזת קידוד
```
שם הסוד:    IOS_DIST_CERT_BASE64
ערך הסוד:   הדבק את מחרוזת הקידוד שנמצאת בלוח ההעתקה
```

הסוד השני — סיסמת הייצוא שבחרת בשלב השלישי
```
שם הסוד:    IOS_DIST_CERT_PASSWORD
ערך הסוד:   הסיסמה שהקלדת בעת יצירת הקובץ הארוז
```

הסוד השלישי — שם הפרופיל, חייב להיות זהה למאת האחוזים לשם שהקלדת בעת יצירת הפרופיל
```
שם הסוד:    IOS_PROFILE_NAME
ערך הסוד:   CarReminder App Store
```

הערה חשובה — הפרופיל עצמו אינו עולה כסוד. במכונה הוירטואלית פעולת ההורדה מתבצעת אוטומטית באמצעות מפתח ה
```
API
```
מהסוד הקיים בשם
```
APP_STORE_CONNECT_API_KEY_BASE64
```

מספיק שהפרופיל קיים בלוח הבקרה של אפל, והוא יורד בכל ריצה של תהליך הבנייה.

## שלב 8 — הפעלה חוזרת של תהליך הבנייה

עבור לעמוד הפעולות בריפו
```
https://github.com/ofek205/CarReminder-to-App/actions
```

בחר את התהליך
```
iOS Release
```

לחץ על
```
Run workflow
```

השאר את שדה מספר הבנייה ריק, ולחץ על אישור. תהליך הבנייה אמור לסיים בהצלחה תוך כעשרים דקות, ובסופו תראה את העלאת החבילה לטסטפלייט.

אם התהליך נכשל — בדוק את שלוש הסודות החדשים, ושים לב במיוחד לכך ששם הפרופיל בסוד זהה למאת האחוזים לשם שבחרת בלוח הבקרה.

```
ios_distribution.key
ios_distribution.p12
```
ושל הסיסמה. הם יידרשו אם תצטרך לחדש את התעודה בעתיד או להפיץ ממחשב מקומי. את שאר הקבצים בתיקיית העבודה ניתן למחוק.
